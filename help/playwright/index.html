🎯 Введение в Playwright

Playwright — мощный фреймворк для автоматизации браузеров (Chromium, Firefox, WebKit). Позволяет эмулировать действия пользователя, проверять состояние страниц, делать скриншоты и многое другое. Ключевой объект — page, предоставляющий API для управления вкладкой браузера.

const { chromium } = require('playwright');
(async () => {
  const browser = await chromium.launch();
  const page = await browser.newPage();
  await page.goto('https://example.com');
  // ... ваши действия
  await browser.close();
})();

Все методы page асинхронны и возвращают Promise. Рекомендуется использовать await или .then().

🧭 Навигация

Основные методы перемещения по страницам и управления историей.

Переход по URL

await page.goto('https://playwright.dev');
await page.goto('https://example.com', { waitUntil: 'networkidle' });

Назад / Вперёд / Перезагрузка

await page.goBack();    // вернуться на предыдущую страницу
await page.goForward(); // перейти вперёд по истории
await page.reload();    // перезагрузить текущую страницу

Опция waitUntil может принимать значения 'load', 'domcontentloaded', 'networkidle', 'commit'.

🔍 Локаторы (поиск элементов)

Современный способ поиска элементов — локаторы. Они автоматически ожидают появление элемента и позволяют выполнять действия.

Методы page.getBy*

// По роли и видимому имени
await page.getByRole('button', { name: 'Отправить' }).click();

// По тексту (точное или частичное совпадение)
await page.getByText('Войти').click();

// По метке (label) и плейсхолдеру
await page.getByLabel('Имя пользователя').fill('admin');
await page.getByPlaceholder('Введите email').fill('user@example.com');

// По test-id (data-testid)
await page.getByTestId('submit-btn').click();

Универсальный locator

// CSS, XPath или текст
const submit = page.locator('#submit');
await submit.click();

const row = page.locator('tr', { hasText: 'Иван' });

Локаторы можно комбинировать: page.locator('article').getByRole('heading').

🖱️ Действия над элементами

Все действия автоматически ожидают, что элемент станет видимым, доступным и не перекрыт другим элементом.

const button = page.getByRole('button', { name: 'Купить' });

// Клик
await button.click();
await button.click({ button: 'right', clickCount: 2 }); // правый, двойной

// Ввод текста (очищает поле перед вводом)
await page.getByLabel('Имя').fill('Анна');

// Выбор из выпадающего списка
await page.locator('select').selectOption('blue');

// Чекбоксы / радиокнопки
await page.getByLabel('Согласен').check();
await page.getByLabel('Не согласен').uncheck();

// Наведение мыши
await page.getByRole('link', { name: 'Подробнее' }).hover();

// Нажатие клавиш
await page.locator('#search').press('Enter');

Также доступны .dblclick(), .focus(), .setInputFiles() для загрузки файлов.

⌛ Явные ожидания

Хотя Playwright имеет встроенные автоматические ожидания, иногда требуются ручные.

// Ожидание появления селектора
await page.waitForSelector('.toast-success');

// Ожидание определённого состояния загрузки
await page.waitForLoadState('networkidle');

// Ожидание смены URL
await page.waitForURL('**/profile');

// Принудительная задержка (только для отладки!)
await page.waitForTimeout(1000);

Рекомендуется избегать waitForTimeout в продакшен‑тестах.

📸 Скриншоты и PDF

// Скриншот всей страницы или элемента
await page.screenshot({ path: 'screenshot.png', fullPage: true });
await page.locator('#header').screenshot({ path: 'header.png' });

// Генерация PDF (только Chromium)
await page.pdf({ path: 'page.pdf', format: 'A4' });

Для скриншотов доступны опции clip, quality (для JPEG), omitBackground и другие.

⚡ Выполнение JavaScript в контексте страницы

// Выполнить функцию и получить результат
const title = await page.evaluate(() => document.title);

// Передача аргументов
const dimensions = await page.evaluate((selector) => {
  const el = document.querySelector(selector);
  return el ? el.getBoundingClientRect() : null;
}, '#main');

// Внедрение скрипта, который будет выполняться при каждой навигации
await page.addInitScript(() => {
  window.myCustomProperty = 'injected';
});

Метод evaluate полезен для получения данных, недоступных через стандартные локаторы.

📡 Отслеживание событий

Можно подписаться на события страницы: консоль, диалоги, запросы и ответы.

// Логи консоли
page.on('console', msg => console.log(`[Browser] ${msg.text()}`));

// Диалоги (alert, confirm, prompt)
page.on('dialog', async dialog => {
  console.log(dialog.message());
  await dialog.accept(); // или .dismiss()
});

// Сетевые запросы
page.on('request', request => console.log('>>', request.method(), request.url()));
page.on('response', response => console.log('<<', response.status(), response.url()));

// Ожидание конкретного запроса
await page.waitForResponse(response => response.url().includes('/api/login'));

Отписка от события: page.off('console', handler).

🧠 Полезные приёмы

Работа с iframe

const frame = page.frame({ url: /iframe-example/ });
await frame.getByText('Кнопка в iframe').click();

Несколько вкладок / контекстов

const context = await browser.newContext();
const page1 = await context.newPage();
const page2 = await context.newPage();

Эмуляция устройств

const iPhone = devices['iPhone 12'];
const context = await browser.newContext({ ...iPhone });
const page = await context.newPage();

Сохранение состояния аутентификации

await context.storageState({ path: 'state.json' });
// позже
const context = await browser.newContext({ storageState: 'state.json' });

Совет Всегда используйте локаторы на основе ролей и текста — они делают тесты устойчивыми к изменениям в DOM.
