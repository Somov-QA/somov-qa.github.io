Вот полный список параметров командной строки для автотестов `playwright test`.

### ⚙️ Общие параметры
* **`test [file...]`**: Запускает указанный файл или тест.
* **`--help`**: Показывает справку по всем командам.

### 🚀 Настройка процесса
*   **`--config/-c <file>`**: Указывает путь к файлу конфигурации (`playwright.config.ts`).
*   **`--workers/-j <workers>`**: Количество параллельных процессов (по умолчанию 50% ядер). `1` отключает параллелизацию.
*   **`--project <browser>`**: Запуск для конкретного браузера (`chromium`, `firefox`, `webkit`, `all`).
*   **`--reporter <reporter>`**: Формат отчета (`line`, `dot`, `json`, `junit`).
*   **`--headed`**: Запуск в режиме с открытым браузером (GUI), а не в фоне.
*   **`--timeout <ms>`**: Таймаут для одного теста в миллисекундах.
*   **`--global-timeout <ms>`**: Общее время на прогон всех тестов.
*   **`--retries <number>`**: Количество повторных прогонов упавших тестов.
*   **`--browser <browser>`**: Запуск в конкретном браузере (`chromium`, `firefox`, `webkit`).
*   **`--trace <value>`**: Настройка сбора трейсов (`on`, `off`, `retain-on-failure`).
*   **`--screenshot <value>`**: Настройка скриншотов (`on`, `off`, `only-on-failure`).
*   **`--video <value>`**: Настройка видео (`on`, `off`, `retain-on-failure`).
*   **`--max-failures <number>`**: Останавливает выполнение после указанного числа провалов.
*   **`--shard <shard>`**: Делит тесты на части (`x/y`) для параллельного прогона.
*   **`--output <dir>`**: Указывает папку для артефактов (результаты, скриншоты).
*   **`--repeat-each <n>`**: Каждый тест запускается `n` раз подряд.

### 🔍 Фильтрация и отбор тестов
*   **`--grep/-g <pattern>`**: Запуск тестов, чьё название или тег совпадает с регулярным выражением.
*   **`--grep-invert <pattern>`**: Запускает тесты, которые **НЕ** совпадают с регулярным выражением.
*   **`--forbid-only`**: Прерывает прогон, если используется `test.only` (полезно для CI).
*   **`--last-failed`**: Перезапускает только упавшие тесты из последнего прогона.
*   **`--repeat-each`**: Запускает каждый тест `N` раз. Полезно для поиска нестабильных тестов.
*   **`--test-list <file>` / `--test-list-invert <file>`**: Запуск или исключение тестов из списка в текстовом файле.
*   **`--fully-parallel`**: Запускает все тесты из разных файлов параллельно, игнорируя настройки файлов.

### 🐛 Отладка и диагностика
*   **`--debug`**: Запускает Playwright Inspector, автоматически включает `--headed --timeout 0`.
*   **`--ui`**: Запуск интерактивного UI для просмотра и отладки шагов.
*   **`--slowmo <ms>`**: Замедляет выполнение действий на `N` мс для визуального контроля.
*   **`--trace <value>`**: Настройка сбора трейсов (`on`, `off`, `retain-on-failure`).
*   **`--ignore-snapshots`**: Игнорирует несовпадения визуальных снэпшотов.
*   **`--list`**: Покажет список тестов в формате JSON, но не запустит их.
*   **`--fail-on-flaky-tests`**: Завершает код ошибкой при обнаружении нестабильных тестов.
