1. Подготовка к работе

libGDX ››

Для работы с libGDX вам понадобится Gradle,JDK, AndroidSDK и редактор. Чтобы упростить себе жизнь советую использовать Android Studio.

1) Скачать Gradle (Complete distribution) https://gradle.org/gradle-download/

1.1) Распаковать архив gradle-3.0-all.zip в папку C:\Program Files (x86)\gradle-3.0

1.2) Добавить переменную среды (Свойство системы → Доп. параметры среды → Параметры среды)

В окне "Системные параметры" создать переменную:

наименование: GRADLE_HOME

значение: C:\Program Files (x86)\gradle-3.0\bin



Так же этот путь при необходимости можно прописать в Path (Переменные среды пользователя)

2) Скачать и установить JDK (Java SE Development Kit 8u101 или новее ) http://www.oracle.com/technetwork/java/javase/downloads/jdk8-downloads-2133151.html

2.1) Добавить переменную среды (Свойство системы → Доп. параметры среды → Параметры среды)

В окне "Системные параметры" создать переменную:

наименование: JAVA_HOME

значение: C:\Program Files\Java\jdk.1.8.0_101



Так же при необходимости можно прописать путь в jdk в переменной среды пользователя Path



3) Скачать и установить Android Studio (версия 2.2.0 или новее) https://developer.android.com/studio/index.html

Если вы собираетесь разрабатывать мобильное приложение вам понадобятся пакеты Abdroid SDK

Для этого запустите SDK Manager который по умолчанию находится по адресу C:\Users\ИмяПользователя\AppData\Local\Android\sdk  или  в Android Studio через меню Tools → Android → SDK Manager

Вам нужно установить свежие версии Tools и Platform-tools. Может случатся так что движок еще не знает о новых версиях и вам необходимо установить предыдущую версию, в моём случае я ставил 23 и 24 версии.



Далее по своему усмотрению нам нужно установить пакеты Android API, я ставил Android 2.2 (API 8) и Android 4.4W2 (API 20) и Android 7.0 (API 24)

А так же дополнительные пакеты драйверов и прочее
